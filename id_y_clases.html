¿Por qué no deberíamos repetir el id en varios elementos?

Un id es un identificador único y solo debería asignarse a un único elemento en toda la página HTML. Si se repite el mismo id en varios elementos,
el documento HTML deja de ser válido y los comportamientos del navegador pueden ser inconsistentes.
Al seleccionar un id con CSS, el navegador generalmente seleccionará solo el primer elemento que encuentre con ese id,
lo que puede hacer que los estilos o scripts no se apliquen correctamente a otros elementos con el mismo id.

¿Qué tiene más peso, un id o una clase en CSS?

En CSS, las reglas con id tienen mayor especificidad que las reglas aplicadas a través de clases.
Es decir, si un elemento tiene tanto una clase como un id, el estilo definido para el id sobrescribirá el estilo definido para la clase,
siempre y cuando ambos estilos modifiquen la misma propiedad.
